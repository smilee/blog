Deploy a Hexo Blog to Your GitHub Pages

Apr 27, 2019 in blog, Hexo

While asking around for recommendations on blogging tools, I came across Jekyll, Hexo, and Gatsby. All of these choices are popular, so it all came down to personal preference. Well, Jekyll is based on Ruby and Hexo and Gatsby on Node.js. Considering that I am more comfortable with JavaScript, I wanted to go with either Hexo or Gatsby. There was one more thing to consider, though - the degree of freedom. I didnâ€™t need the freedom Gatsby offered, and before long, I had this blog up and running.

Installing Hexo

Installation was a breeze because I already had the requirements installed.

Requirements

  • Node.js
  • Git

Once you have the requirements installed, simply run the following command on the command line to install Hexo:

$ npm install -g hexo-cli

Initializing Hexo

Once Hexo is installed, run the following commands in the target folder:

$ hexo init <folder>
$ cd <folder>
$ npm install

Installing Plugins

Install hexo-deployer-git plugin:

$ npm install hexo-deployer-git --save

Configuring Hexo

Find and open _config.yml to add the following adjustments:

deploy:
  type: git
  repo: https://username.github.io/blog
  branch: gh-pages
  message:

Initializing Git

Initialize Git by running the following command in the command line:

$ git init

Creating a GitHub Repository

Visit and log into GitHub and make a new repository. Name the repository, check the public option, then click on Create repository.

Adding a New Remote

Take the remote URL from GitHub and put it in the following command as shown below:

$ git remote add origin git@github.com:username/blog.git

Generate and Deploy

Now you can deploy your sample post by running the following commands:

$ hexo generate
$ hexo deploy

Voila!
