React 기초: 소개

May 15, 2019 in javascript, react

React를 시작하는 분을 위해 간단한 소개를 준비했습니다.

관심사의 분리

  • React
    • 유저 인터페이스를 만들기 위한 JavaScript 라이브러리
    • 컴포넌트와 컴포넌트가 어떻게 동작하는지 정의한다.
  • ReactDOM
    • React를 위한 JavaScript 패키지
    • React 엘리먼트를 DOM에 렌더링하는것과 관련된 부분을 담당한다.
    • React 모델 밖으로 나가기 위한 비상구 역할을 한다.

위에서 React가 관심사의 분리를 통하여 다양한 플랫폼에서 쓰기 편한 구조로 설계되어있음을 알 수 있다.

시작하기

리액트를 사용하는 방법은 크게 두가지가 있다.

  1. 웹사이트에 React 추가
    • 간편한 맛보기에 적합하다.
  2. 툴체인 사용
    • 배우는 단계이거나 SPA를 만들 때: Create React App
    • Node.js로 SSR이 되는 웹사이트를 만들 때: Next.js
    • 정적 웹페이지가 주인 웹사이트를 만들 때: Gatsby
    • 컴포넌트 라이브러리 작업이나 기존 코드베이스 위에 작업할 때: Neutrino, nwb, Parcel, Razzle 등
    • 툴체인을 직접 만들어서 사용
      • 흔히 패키지 매니저(Yarn 또는 npm), 번들러(webpack 또는 Parcel), 그리고 컴파일러(Babel)의 조합으로 만든다.
      • 당신이 이것이 가능한 수준이라면 그만 읽고 어서 코드를 짜는것을 추천한다.

패키지 매니저나 번들러는 패키지를 사용할 때 편하다면, Babel은 ES6이상의 JavaScript 코드를 대부분의 브라우저가 이해할 수 있는 코드로 변환해주기 위함이다. 물론 어떤 이는 ES5를 쓰면 되지 않냐고 물을 수 있다. 하지만 왜 그런 제한을 둬야만 하는가?

Create React App

이 중 흔히 쓰이는 Create React App이라는 툴체인을 사용하여 프로젝트를 만드는 경우 생성되는 프로젝트 구조는 다음과 같다.

  • README.md
    • 프로젝트 목표 및 사용법 등 설명
  • node_modules
    • 프로젝트가 사용하는 패키지가 들어있다.
    • 용량이 커서 .gitignore에 이 폴더를 추가하는 경우가 흔하다.
  • package.json
    • 프로젝트 정보
    • node_modules 폴더가 없어도 프로젝트가 사용하는 패키지를 알기 때문에 각자 직접 다움로드받기 용이하게 한다.
  • .gitignore
    • 일부러 신경쓰지 않는 파일목록을 관리한다.
  • public
    • 정적 파일 (이미지 등)을 담아두는 공간이다.
  • src
    • 소스코드를 담아두는 공간이다.

이렇게 생성된 프로젝트는 사용하는 패키지 매니저에 따라 npm start 또는 yarn start 명령을 통해 개발모드로 실행할 수 있다. 이 프로세스는 ctrl + c 명령으로 종료할 수 있다. 보다 상세한 설명은 공식 README.md에서 확인하자.

모듈 시스템

앞서 ES2015(ES6)를 사용해도 브라우저가 알아들을 수 있도록 Babel을 쓴다고 했다. 하지만 필요한 패키지를 찾아서 쓰다보면 가끔 [...] has no default export이라는 에러메시지를 만날 때가 있다. 이는 그 패키지가 다른 모듈 시스템(CommonJS)을 사용하고 있어서이다. 현재 Babel이 import와 export를 컴파일 할 수 있고, Node.js v12.2.0에서는 import가 지원된다. 그리고 CommonJS 모듈 시스템을 사용하는 패키지는 관리가 안되고 있을 가능성이 높고 그 와중에 비슷한 기능의 다른 패키지가 생겼을 가능성이 높으므로 대안도 고민해보면 좋다.

  • CommonJS 모듈 시스템 (Node.js)
    • require (node_modules에서 해당 모듈을 찾는다)
    • module.exports
  • ES6 모듈 시스템
    • import
    • export

리액트 컴포넌트는

  • 함수 또는 클래스다.
  • 사용자에게 보여줄 HTML을 JSX로 만든다.
  • 사용자의 피드백을 이벤트 핸들러를 통해 처리한다.
